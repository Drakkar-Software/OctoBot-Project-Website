Ne faites pas confiance, vérifiez

OctoBot est à l’image des cryptomonnaies: son fonctionnement est entièrement transparent.

Quelle est la valeur d'un investissement ou d'un trade si il vous empêche de dormir par peur d'une mauvaise décision? Comment s'assurer que le robot se comportera comme prévu ? La philosophie d'OctoBot est d'être le plus transparent et fiable possible.

  • Le code est open source afin de pouvoir vérifier ce que fait le robot.
  • les stratégies sont testables sur les données du passé et les données en temps réel pour garantir la fiabilité des stratégies.
Résultat de backtesting

Développé pour la performance

Optimisation sur plusieurs niveaux

OctoBot est architecturé pour être très rapide et adaptatif tout en laissant le développement d'extensions abordable pour un développeur débutant. Pour accomplir ses objectifs, le robot est developpé en Python selon une architecture asynchrone avec asyncio permettant d'optimiser le temps CPU. De plus, les procédures coûteuses en puissance machine ou en mémoire sont exécutées en code natif à la plateforme et optimisé grâce à Cython. Grâce à cette architecture, OctoBot est tout à fait capable de fonctionner à 100% sur des environnements minimaux tels d'une Raspberry Pi.
